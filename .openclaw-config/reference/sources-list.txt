# 采集策略

## 优先级排序
1. 官方博客（100% 采集）
2. 技术媒体（80% 采集）
3. 学术论文（50% 采集，仅重要论文）
4. 开发者社区（30% 采集，仅高分帖子）

## 采集频率
- 官方博客：每次都采集
- 技术媒体：每次都采集
- 学术论文：每天采集一次
- 开发者社区：每天采集一次

## 社区参与度采集
- Hacker News：使用 HN API 获取评分和评论数
- Reddit：使用 Reddit API 获取投票和评论数
- GitHub：使用 GitHub API 获取 Star 数

## 时间范围
- 优先采集最近 24 小时的内容
- 最多回溯 48 小时

# 使用方法

AI Agent 应该：

1. **读取这个文件**，获取所有信源 URL
2. **按优先级顺序**采集每个信源
3. **使用 WebFetch 工具**获取页面内容
4. **提取关键信息**：
   - 标题
   - 摘要/简介
   - 链接
   - 发布时间
   - 作者（如果有）
5. **保存到 data/sources_raw.json**

示例代码（伪代码）：

```python
sources = read_file("reference/sources-list.txt")
articles = []

for source in sources:
    content = web_fetch(source.url)
    extracted = extract_articles(content)
    articles.extend(extracted)

save_json("data/sources_raw.json", articles)
```

# 注意事项

1. **仅使用英文信源**，不采集中文媒体
2. **尊重 robots.txt** 和 rate limiting
3. **缓存结果**，避免重复采集
4. **记录失败的信源**，便于后续排查
5. **验证链接有效性**，过滤失效链接
